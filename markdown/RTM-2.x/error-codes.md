本文介绍在使用 RTM 2 APIs 时 SDK 可能会返回的错误码。

RTM 2 SDK 在运行过程中，在方法调用失败的返回值中，返回一个负数，这个负数就对应着错误码里的正数。具体错误码详见以下表格：


| 枚举值   | 错说明      | 
| ------------ | --------- |
| `10001`     | Topic 已存在。  | 
| `10002`     | 创建的 Topic 超出数量限制。  | 
| `10003`     | 不是有效的 Topic 名称。  | 
| `10004`    | 发布 Topic 失败。  | 
| `10005`    | 订阅的 Topic 超出数量限制。  | 
| `10006`    | 频道内用户超出数量限制。  | 
| `10007`    | 创建的频道超出数量限制。  | 
| `10008`    | 已加入该频道。  | 
| `10009`    | 未加入任何频道。  |
